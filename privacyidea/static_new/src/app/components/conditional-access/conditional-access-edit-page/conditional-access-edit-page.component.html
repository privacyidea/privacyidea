Priority decides how this policy ranks against the others: for an allow/deny decision the highest-priority matching policy wins, while lock, block and email policies all run regardless of priority.

Stages & actions

A stage fires when the tracked events reach its failure threshold within the time window, and then runs its actions (lock the user, block the IP, send an email, allow or deny). Add several stages to escalate — but only one stage of this policy fires per evaluation: when several thresholds are met at once, only the stage with the highest met threshold runs and the rest are skipped. Other policies are evaluated on their own (see the policy Priority above).
