# Angular Dependency Updates

## Background

The Angular application is located in `privacyidea/static_new/` and uses npm as the package manager.

**package.json**

The `package.json` file is the manifest for the angular project. It contains:
- **Project metadata**: Name, version, description, and scripts
- **Dependencies**: Production packages required for the application to run (e.g., `@angular/core`, `rxjs`)
- **DevDependencies**: Development-only packages needed for building and testing (e.g., `@angular/cli`, `jest`, `typescript`)
- **Version ranges**: Uses semantic versioning with prefixes like `^` (compatible updates) or `~` (patch updates)

**package-lock.json**

The `package-lock.json` file is automatically generated by npm and serves as a lockfile that:
- **Records exact versions**: Locks down the precise version of every package installed, including nested dependencies
- **Ensures consistency**: Guarantees that all developers and CI/CD systems install identical dependency trees
- **Tracks integrity hashes**: Contains cryptographic hashes to verify package integrity


## Vulnerability Detection

You can check for known vulnerabilities in your dependencies running `npm audit`.
This will display a report of vulnerabilities with severity levels (low, moderate, high, critical).


## Updating Vulnerable Packages

You can update all vulnerable packages by running `npm audit fix`. 
To enforce updates including breaking changes, use `npm audit fix --force`.

Single packages can be updated with `npm update <package-name>` or `npm install <package-name>@<version>`.


## Updating All Dependencies

### Minor Updates with npm update

Update all packages within their semver ranges (respects `^` and `~` in package.json) using `npm update`. 
This also updates the `package-lock.json`.


### Update Angular

Angular provides its own update tool `ng update`. It updates `package.json` with new Angular package versions and 
`package-lock.json` with the locked versions, installs packages into `node_modules` and runs migrations if necessary.

Major versions can only be updated one by one. Also check the Angular update guide for any manual steps required for 
the new version: https://angular.dev/update-guide.

1. Update Node.js to the version required by the new Angular version (or the latest version if supported):
   ```bash
   nvm install <version>
   nvm use <version>
   ```

2. Update Angular core and CLI:
   ```bash
   ng update @angular/core@20 @angular/cli@20
   ```
3. Update Angular Material:
    ```bash
   ng update @angular/material@20
   ```
4. Update Angular dev dependencies:
   ```bash
   ng update @angular-builders/jest@20
   ```


### Update Remaining Dependencies

While angular should always be updated first using the `ng update` command, other dependencies can be updated using 
`npm-check-updates` (ncu) and `npm install`.
With `ncu -u` versions in the `package.json` will be updated to the latest versions. Running `npm install` afterward 
updates `package-lock.json` and installs new dependencies.

Useful options for `ncu`:
- `-u` - Update `package.json` with new versions (without this option, ncu only checks for new versions and prints them to the console)
- `--target semver` - Only update according to semver rules (e.g. minor updates only)
- `-c 7d`- Only packages with a minimum age of 7 days are updated to reduce the risk of supply chain attacks

```bash
npm install -g npm-check-updates
ncu -u --target semver -c 7d
npm install
```


### Troubleshooting

**Dependency conflicts:**
```bash
# Clear cache and reinstall
rm -rf node_modules package-lock.json
npm cache clean --force
npm install
```
